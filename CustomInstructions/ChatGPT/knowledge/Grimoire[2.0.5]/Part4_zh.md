## 第 13 章：诅咒，Cursor.sh 101
40：File > New Ai project
在左上角，打开 file，隐藏在底部附近的不起眼位置是：New Ai Project
写下你的提示词（确保把提示词保存在其他地方，以防你想重新生成和调整，这个屏幕后它会消失）。"用 html、css 和 js 编写 pong"
选择一个文件夹来创建它
坐下来看你的应用通过多文件生成开始

这个体验相当有 bug
小心不要点击到不同的文件并中断流式传输，否则会出问题
它有时也会随机卡住。如果发生这种情况，就重新开始。（你之前保存了提示词对吧？）

我喜欢同时做 2-3 个这样的，比较它们以创建一个起始基础。我通常会在之后创建一个新的干净项目，然后挑选部分来组成我的实际项目。仅将这些用于灵感和起点

41：侧边聊天，Cmd + Shift + L
用 CMD + Shift + L 打开 cursor 侧边聊天窗口
你可以用它与你的整个代码库聊天！
默认情况下它搜索你当前的文件，但你也可以用 cmd+enter 搜索整个项目
你可以高亮代码部分并按 CMD + SHIFT + L 将它们添加到聊天中

42：Cmd + K
在编辑器中，点击 cmd+k 打开快速命令面板
然后简单地要求你想要它写的内容并按生成
不骗你，我现在可能用这个热键写了超过 50% 的代码。它就是这么好。

你会看到创建的 diff。接受或拒绝
注意你可以跨此撤销/重做，这超级方便

你可以对此变得相当随意。有时你只需要说"继续"或"你知道该怎么做"，它足够聪明，可以阅读周围的上下文

你也可以提交后续请求来帮助完善它

有时我会生成，复制它，重新生成，再复制，再重新生成，再复制，然后粘贴粘贴粘贴，只是为了获得一些自己编写的想法

这对于编辑也超级方便！！！

选择一块文本，然后 cmd+k，然后要求你想要如何编辑它
"把这个按钮变成蓝色"
"添加注释"
"简化"
"拆分成更小的组件"
"重命名 ..."

43：终端中的 Cmd + K
注意，如果你正在运行和调试你的应用，cmd+k 在 cursor 终端窗口中也可用。使用 cmd+K 生成终端命令或 cmd + / 自动完成
