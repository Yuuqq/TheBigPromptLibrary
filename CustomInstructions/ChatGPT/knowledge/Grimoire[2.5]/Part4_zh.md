## 第13章：诅咒，Cursor.sh 入门
40: File > New Ai project
在左上角，打开 file，在底部附近隐藏着不起眼的：New Ai Project
写你的提示（确保将提示保存在其他地方，以防你想重新生成和调整，它会在这个屏幕后消失）。"用 html、css 和 js 写 pong"
选择一个文件夹来创建它
坐下来看着你的应用通过多文件生成开始

这个体验相当有问题
小心不要点击到其他文件并中断流式传输，否则会中断
它有时也似乎随机卡住。如果发生这种情况，就重新开始。（你之前保存了你的提示对吧？）

我喜欢同时做2-3个这样的，并比较以创建一个起点。我通常会在之后创建一个新的干净项目，挑选和选择要组合到我实际项目中的部分。只用这些作为灵感和起点

41: 侧边聊天，Cmd + Shift + L
用 CMD + Shift + L 打开 cursor 侧边聊天窗口
你可以用它与你的整个代码库聊天！
默认情况下它搜索你当前的文件，但你也可以用 cmd+enter 搜索整个项目
你可以高亮代码部分并按 CMD + SHIFT + L 将它们添加到聊天中

42: Cmd + K
在编辑器中，按 cmd+k 打开快速命令面板
然后简单地询问你想让它写什么并按 generate
老实说，我现在可能用这个热键写了超过50%的代码。它太好用了。

你会看到一个 diff 被创建。接受或拒绝
注意你可以在此过程中撤销/重做，这非常方便

你可以用这个变得有点傻。有时你只需要说"continue"或"you know what to do"，它就足够聪明来阅读周围的上下文

你也可以提交后续请求来帮助完善它

有时我会生成，复制它，重新生成，再复制，重新生成，再复制，然后粘贴粘贴粘贴，只是为了得到一些自己写的想法

cmd+K 对于编辑也超级方便！！！

选择一块文本，然后 cmd+k，然后询问你想如何编辑它
"make this button blue"
"add comments"
"simplify"
"break out into smaller components"
"rename ..."

43: 终端中的 Cmd + K
注意，如果你正在运行和调试你的应用，cmd+k 也可以在 cursor 终端窗口中使用。使用 cmd+K 生成终端命令或 cmd + / 自动完成

## 第14章：咒术，Cursor.sh 进阶
44: 侧边聊天第2部分，@codebase，高级搜索，手动 RAG，.cursorIgnore
使用 cursor 侧边聊天时。默认情况下，提问会对当前打开的文件进行 RAG。
你也可以使用 cmd+enter 来"chat with codebase"以搜索和 RAG 多个文件。
这对于连接代码库不同部分的想法非常有用。

使用"final codebase context"下拉菜单查看 AI 正在读取的文件片段。

使用"advanced"下拉菜单手动控制 RAG。你可以更改文件数量。包含或排除某些文件类型。选择块重排序方法，添加可选的额外推理步骤

我强烈建议在你的项目中创建一个 .cursorIgnore 文件！！
类似于 .gitIgnore，这是一个你想在使用 cursor 侧边聊天时忽略的文件列表。
非常方便，确保愚蠢的配置文件不会干扰 ai 的思考。
