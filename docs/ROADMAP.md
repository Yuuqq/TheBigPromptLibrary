# 一年迭代路线图 (2026 Q3 — 2027 Q2)

> 本文档由维护者制定，描述 [TheBigPromptLibrary 中文版](https://github.com/Yuuqq/TheBigPromptLibrary) 在未来 12 个月的发展方向。最近一次更新：2026-05-02。

## 🎯 一年愿景

> **从"上游英文提示词的中文镜像站"升级为"中文 LLM 提示词工程的事实标准入口"**

不仅是翻译跟同步，还要在 **搜索发现 / 内容智能化 / 社区贡献** 三个维度建立独有价值。

## 📊 北极星指标

| 维度 | 当前 | 12 个月目标 |
|---|---|---|
| 翻译覆盖率 | ~100% | ≥99%（持续保持） |
| 平均翻译质量分 | 待首次跑出 | ≥88 / 100 |
| 月独立访客 | 未知 | 5,000+ |
| 中文原创提示词 | 0 | 100+ |
| GitHub Star | 0 | 200+ |
| 平均会话深度 | 1-2 页 | 4+ 页 |

---

## Q3 2026（5–7 月）— 基础硬化季

**主题**：把已有自动化做稳，修复早期翻译质量遗留。

| # | 任务 | 价值 | 工作量 |
|---|---|---|---|
| 3.1 | **Diff-based 翻译**：上游文件更新时只翻译变化段落 | 省 70% token、加速 3x | 3d |
| 3.2 | **翻译记忆库**：缓存常见短语/段落 hash，跨文件复用 | 一致性 +30% | 5d |
| 3.3 | **多模型容错**：GLM 挂了自动降级 Qwen/DeepSeek | 99.9% SLA | 2d |
| 3.4 | **重译 worst-50**：质量分 <70 的全部用 GLM-4-Air 重做一遍 | 整体质量 +5 分 | 1d（自动） |
| 3.5 | **术语一致性检查器**：扫全库找"prompt"等被译成不同词的位置 | 用户阅读体验 | 2d |
| 3.6 | **翻译纠错 PR 模板**：用户发现错译 → 一键提交修正 PR | 启动社区 | 1d |

**季末验收**：覆盖率 ≥99%、平均质量 ≥85、连续 30 天 0 宕机。

## Q4 2026（8–10 月）— 发现层季

**主题**：让 1849+ 提示词不再被埋没。

| # | 任务 | 价值 | 工作量 |
|---|---|---|---|
| 4.1 | **全文搜索**：lunr.js 索引正文前 500 字（不只是标题/摘要） | 找到率 3x | 3d |
| 4.2 | **AI 自动打标签**：LLM 给每个提示词打 5-10 个语义标签 | 浏览效率 +50% | 4d |
| 4.3 | **相似提示词推荐**：基于标签 + Jaccard 相似度 | 粘性 +30% | 2d |
| 4.4 | **上游 diff 视图**：标记"自上次访问起新增/变化"的提示词 | 老用户回访 | 3d |
| 4.5 | **并排对比**：同时打开两个提示词对比差异 | 进阶用户 | 3d |
| 4.6 | **移动端重构**：响应式优化 + 触摸手势 + PWA 离线 | 移动流量 +200% | 5d |

**季末验收**：移动端可用、平均会话深度 4+、自动标签覆盖 100%。

## Q1 2027（11 月–1 月）— 智能化季

**主题**：用 AI 让每条提示词更易理解、更易使用。

| # | 任务 | 价值 | 工作量 |
|---|---|---|---|
| 5.1 | **LLM-as-judge 质量评分**：替换启发式打分，对样本用 Claude/GPT-4o 评分 | 质量评估更准 | 4d |
| 5.2 | **AI 摘要**：每条提示词自动生成 1 句话 + 1 段中文导读 | 新用户秒懂 | 3d |
| 5.3 | **用法示例生成**：为每个 system prompt 自动产生 3 个 user 输入示例 | 直接可复制 | 4d |
| 5.4 | **语义搜索**：用 BGE 中文 embedding，存 384 维向量 | 模糊查询 | 5d |
| 5.5 | **"拆解此提示词"按钮**：调 LLM 生成结构分析 | 学习价值 | 3d |
| 5.6 | **风险分级**：自动标注 `safe / sensitive / jailbreak` | 教育用户 | 2d |

**季末验收**：智能摘要 100% 覆盖、语义搜索可用、AI 评分与人工 ±5 分。

## Q2 2027（2–4 月）— 社区季

**主题**：从镜像站升级为有原创内容的社区。

| # | 任务 | 价值 | 工作量 |
|---|---|---|---|
| 6.1 | **中文原创提示词区**：新建 `Prompts_zh/` 目录 + PR 模板 | 独有价值 | 3d |
| 6.2 | **匿名投票**：每条 👍/👎，用 Cloudflare Worker + KV 存 | 社交证明 | 4d |
| 6.3 | **每周精选自动 Issue**：周一 Action 跑 LLM 选出本周精选 | 用户回访 | 2d |
| 6.4 | **提示词工程教程区**：5 篇中文长文（基础/进阶/越狱/CoT/agent） | SEO + 留存 | 10d |
| 6.5 | **多语言扩展（日韩）**：一键 fork 模板 | 生态延展 | 5d |
| 6.6 | **RSS / 微信通知机器人**：新增内容推送 | 复访渠道 | 3d |

**季末验收**：100+ 原创、平均每周 3 个外部 PR、RSS 订阅 200+。

## 🛡️ 风险与预案

| 风险 | 概率 | 影响 | 预案 |
|---|---|---|---|
| GLM 免费档突然收费/下线 | 中 | 高 | Q3.3 多模型容错先做；备好 DeepSeek/Qwen API key |
| 上游 0xeb 仓库停更或反对镜像 | 低 | 高 | 提前邮件沟通；保留独立内容（Q2 原创区做缓冲） |
| GitHub Pages 流量超 100GB/月 | 低 | 中 | Q4 加 jsDelivr CDN for `prompts_index.json` |
| LLM 翻译被检测出大量低质 | 中 | 中 | Q3.1 + Q3.4 + Q1.1 三道防线 |
| GitHub Actions 配额耗尽 | 低 | 低 | 工作流时间 < 10 分钟，月度配额 2000 分钟 ≫ 需要 |
| 法律风险（提示词版权） | 低 | 高 | Q3 设立 LICENSE 审计 + DMCA 流程 |

## ⏸️ 已知不做项（保持聚焦）

- ❌ 用户登录系统（隐私 + 维护成本太高，用 localStorage 已足够）
- ❌ 评论系统（用 GitHub Discussions 替代）
- ❌ 自建后端服务（保持 GitHub Pages 静态站架构）
- ❌ 移动 App（PWA 已经够用）
- ❌ 商业化 / 付费墙（保持开源精神）
- ❌ 实时聊天 / Discord 机器人（资源不够运维）
